## 3. 노트 표준 포맷 (체크 시스템의 핵심)

모든 개념 노트 최상단 frontmatter:

```yaml
---
tags: [운영체제]
status: 안함          # 안함 → 공부중 → 완료 → 복습
priority: 1           # 면접 빈도 1(빈출) ~ 3(심화)
복습일:               # (선택) 다음 복습 예정일
출처: gyoogle
---
```

본문: 개념 설명 + 관련 개념으로의 `[[위키링크]]`.

→ 공부 여부 추적은 **`status` 필드 하나**로 관리. 공부하면 값만 바꾸면 대시보드/그래프가 자동 갱신.

## 4. 학습 로드맵 (면접 빈도 우선 — "무엇부터")

대시보드의 "다음에 할 것"은 이 Tier 순서를 따른다.

- **Tier 1 (빈출 · 가장 먼저)**: 프로세스/스레드, TCP·3-way handshake, HTTP/HTTPS,
  DB 인덱스, 트랜잭션·ACID, 정규화, 가상메모리, 데드락, GC, 시간복잡도, 정렬, 해시/트리
- **Tier 2 (중요)**: 스케줄링, 페이징/세그멘테이션, 락·동시성, 캐시, OSI 7계층,
  쿠키/세션/JWT, 조인, 그래프 알고리즘, DP
- **Tier 3 (심화)**: 디자인패턴 상세, 고급 알고리즘, 기타

→ 각 개념 노트의 `priority` 필드(1/2/3)가 이 Tier에 매핑된다.

## 6. 위키링크(그래프 연결) 생성 방식

**채택: A안 (용어사전 자동 링크) + 추후 C안 선택 적용**

- **A안 (채택)**: CS 핵심 용어 ~150개 사전(glossary)을 구축 → 각 노트에서 해당 용어 첫 등장을
  `[[ ]]`로 자동 변환 + 기존 상대경로 링크(`[txt](../OS/x.md)`)도 `[[ ]]`로 변환.
  빠르고 그래프가 즉시 빽빽해짐. 큐레이션된 용어집으로 오탐 최소화.
- **C안 (추후 정제)**: 빈출(Tier 1) 노트에 한해, 의미 기반으로 링크를 LLM이 보강.

> B안(전 수동)은 "오늘 골격 완성" 목표와 충돌하므로 미채택.

## 7. 구현 컴포넌트 (A단계 = 자동 골격)

1. **소스 확보**: gyoogle 레포 clone → 코어 CS 폴더(Computer Science/Algorithm/Design Pattern)만 추출
2. **변환 스크립트** (재실행 가능):
   - 한글 파일명 정리 + 카테고리 폴더로 이동
   - frontmatter 주입 (status=안함, priority=빈도맵 적용)
   - 용어사전 기반 위키링크 삽입 + 기존 링크 변환
3. **INDEX 3종 작성**: Home(MOC), 진도 대시보드(Dataview), 학습 로드맵(Tier)
4. **템플릿**: `_templates/개념노트.md` (새 노트 추가용)
5. **Obsidian 설정**: `.obsidian/`에 Dataview 플러그인 활성화

### 단계 구분
- **A단계 (이번)**: 위 1~5 → 작동하는 그래프 vault 완성 (오늘 목표)
- **B단계 (추후)**: 긴 문서 원자 노트 분할, 잘못된 링크 정리, 중복 제거 (반복)
- **C단계 (영구)**: 본인이 새 개념 추가하며 확장
