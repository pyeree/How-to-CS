# CS 학습 시스템 강화 설계 (2026-06-05)

전수받은 "CS 면접/시험 학습 지도" 팁을 How-to-CS vault에 반영한다. 팁의 80%(주제 지도·빈출 우선순위·위키링크 그래프)는 이미 구현돼 있고, 비어 있던 3개 갭을 메운다.

## 제약 (먼저 파악)

- `_build/convert.py`는 매 실행 시 카테고리 폴더(`01_*`~`08_*`)를 `shutil.rmtree`로 **통째로 지우고** `_source`에서 재생성한다. → 노트에 손으로 적은 내용은 빌드 때 증발한다.
- `_build/build_index.py`가 `📊 진도 대시보드`·`🗺️ 학습 로드맵`·`_templates/개념노트.md`·MOC·Home을 **생성**한다. → 산출물 `.md`를 직접 편집하면 다음 빌드에 덮어쓰인다.
- 결론: 손댈 곳은 **산출물이 아니라 두 생성기 스크립트**다.

## 갭 1 — 노트 틀 표준화 (30초 요약)

- gyoogle 원문(산문)은 보존하고, 그 위에 `면접 30초 요약` 섹션만 얹는다.
- 보존 마커로 감싼 **수동 영역**을 둔다:
  ```
  <!-- 🔒 MANUAL:START -->
  ## 🎤 면접 30초 요약
  ...
  <!-- 🔒 MANUAL:END -->
  ```
- `convert.py`: rmtree 직전에 기존 노트에서 MANUAL 블록을 스냅샷 → 재생성 후 같은 파일명 노트에 재주입. 추가로 `priority:1` 노트에는 빈 스캐폴드를 자동 삽입(별도 마이그레이션 스크립트 불필요).
- `build_template()`: 템플릿에도 같은 마커 섹션 추가.

## 갭 2 — Claude 학습 루프 자산화

- `build_index.py`에 `build_loop_guide()` 추가 → `00_INDEX/🤖 Claude 학습 루프.md` 생성.
- 복붙용 프롬프트 4종: **채우기 / 출제자 모드 / 꼬리질문 / 연결 찾기**. 연결 찾기 출력은 `[[제목]]` 형식 강제(그래프 엣지로 직결).
- Home에서 가이드 노트로 링크.

## 갭 3 — 복습 소환 (경량 SRS)

- 규칙: 노트를 `완료`로 바꿀 때 `복습일`에 "오늘+3일"을 적는다(가이드 노트에 명시).
- `build_dashboard()`에 "⏰ 오늘 복습할 것" Dataview 추가: `WHERE 복습일 AND 복습일 <= date(today)`.
- 풀 Anki 알고리즘은 YAGNI.

## 손대는 파일

- `_build/convert.py` — MANUAL 블록 보존 + priority:1 스캐폴드
- `_build/build_index.py` — 템플릿/대시보드 갱신 + 가이드 노트 생성 + Home 링크
- (산출물은 빌드로 재생성: 대시보드·템플릿·가이드·priority:1 노트들)

## 알려진 한계

- 카테고리 폴더에 손으로 새로 만든(=`_source`에 대응 원본이 없는) 노트는 다음 `convert.py`에서 삭제된다. 이번 범위 밖(기존 설계의 특성). MANUAL 보존은 원본이 있는 노트 한정.
